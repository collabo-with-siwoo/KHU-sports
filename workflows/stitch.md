# Stitch 디자인 워크플로우

사용법: `/stitch` 또는 "스티치로 디자인 만들어줘"

## 대상 화면 (PRD §4 기준)

### 유저 뷰
- 메인홈 (최신 공지 + 다가오는 대회 + 최근 결과)
- 회원가입 (이메일/비밀번호 + 약관 동적 동의)
- 로그인 / 비밀번호 재설정
- 공지 목록·상세 (페이지네이션, 검색, 카테고리)
- 대회 결과 목록·상세 (순위·이름·총타수)
- 마이페이지 (정보 수정, 비밀번호 변경, 탈퇴)
- 마이페이지 — 내 기록 (스코어카드, 누적 승률)

### 어드민 뷰
- 대시보드 (가입자·공지·다음 대회 요약)
- 공지 관리 (Tiptap WYSIWYG 에디터)
- 회원 관리 (목록·검색·휴면·탈퇴)
- 대회 관리 (일정/장소/종목/라운드)
- 스코어 관리 (수기 + 엑셀 업로드)
- 약관 관리 (추가/수정/비활성/버전 발행)
- 관리자 관리 (초대·권한 부여)

## 워크플로우

### 1. 디자인 요청 확인
- 사용자에게 어떤 화면을 디자인할지 확인
- 필요한 정보 수집:
  - 화면 종류 (위 목록 중 선택)
  - 스타일 (다크/라이트, 미니멀/리치)
  - 반응형 고려 (모바일 375px ~ 데스크톱)

### 2. Stitch MCP로 디자인 생성
- 수집된 정보를 영문 프롬프트로 변환
- StitchMCP 도구를 사용하여 디자인 생성
- 결과 확인

### 3. 사용자 피드백
- 결과를 사용자에게 보여줌
- 수정이 필요한 부분 확인
- 필요 시 2번으로 돌아가서 반복

### 4. 코드 반영 (선택)
- 생성된 디자인을 Next.js 컴포넌트로 변환
- `src/app/` 또는 `src/components/`에 적용
- 커밋 및 push
